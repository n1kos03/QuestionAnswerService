# QuestionAnswerService тестовое задание

## Запуск проекта
Чтобы запустить проект достаточно команды `docker-compose up` можно с флагом `--build` при перезапуске, чтобы образы заново были собраны. Так же можно запустить проект с помощью команды `make`, для запуска тестов `make test`

## Описание проекта
В рамках задания необходимо было построить небольшой микросервис, в котором реализовано API для управления вопросами и ответами на них.

### Архитектура
Проект состоит из директорий:

1) api - в котором содержатся директории с реализацией обработки маршрутов
2) cmd - в которой содержится main функция
3) common - в которой содержится директории с БД, миграциями и моделью данных

### Требования
Для написания запросов использовался чисты net/http

Для управления БД(PostgreSQL) использовался gorm

Миграции выполнены  с помощью goose

Сервис собирается через Dockerfile и запускается с помощью docker-compose

Тестирование API проводилось с помощью Postman

Так же написано немного тестов
